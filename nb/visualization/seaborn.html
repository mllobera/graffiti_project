Seaborn is a library for making statistical graphics in Python. It is built on top of Matplotlib (the main visualization package on which Pandas’s plots are built) and closely integrated with pandas data structures. Seaborn provides a high-level interface to Matplotlib.

Seaborn’s strengths are:

  • Using default themes that are aesthetically pleasing.

  • Setting custom color palettes.

  • Making attractive statistical plots.

  • Easily and flexibly displaying distributions.

  • Visualizing information from matrices and DataFrames.

These last three points are why Seaborn is our tool of choice for Exploratory Analysis. It makes it very easy to “get to know” your data quickly and efficiently.Seaborn aims to make visualization a central part of exploring and understanding data. Think of Seaborn as a complement, not a substitute, to Matplotlib. In general, start with Seaborn to generate a quick plot and then use Matplotlib in you want to tweak it.

Matplotlib is very comprehensive and powerful. Any attribute of the figure can be changed to your liking. A combination of seaborn’s high-level interface and matplotlib’s deep customizability will allow you both to quickly explore your data and to create graphics that can be tailored into a publication quality final product.

Recently, Seaborn has incorporated a new object interface to build charts. So currently, we have the function (older) and the (new) object interfaces. The latter is analogous to R’s ggplot() grammar of graphics style interface.

# New object Interface 
import seaborn.objects as so
from seaborn import axes_style

(
    so.Plot(data= pokemon, x='Sp. Atk', y='Sp. Def', color='Stage', pointsize='Stage', marker='Stage')
      .add(so.Dot())
      .scale(pointsize=(5,10), color='Set1', marker=(['o','X','s']))
      .layout(size=(10,8))
      .limit(x=(40,80))
      .theme(axes_style('whitegrid'))
)

     .add(so.Dash(color='r', width=0.25), so.Agg('median'))

Figure level plots#

So Figure level plots as opposed to axes plots are used to generate multiple plots very quickly. The former simplify the creation of multiple axes plots. The following demonstrate the ability of Seaborn to generate lots of output by ‘mapping’ multiple attributes to various chart properties and dimensions.

Relational Plot relplot()#

Used to explore the relationship between two variables. It extends the following axes plots, that is, it will ‘call’ one of the following repeatedly in order to create the final plot:

  • scatterplot() (with kind=“scatter”; the default)

  • lineplot() (with kind=“line”)

# Relational Plot 1) size = 'Speed', 2) sizes = (30, 100), 3) row = 'Stage', 4) col = 'Speed Cat' 5) kind= 'line' comment out size and sizes
sns.relplot(
    data= pokemon,
    x = 'Attack',
    y = 'Defense',
    style = 'Stage',
    hue = 'Type 2',
    size = 'Speed',
    sizes = (30, 100),
    row = 'Stage',
    col = 'Speed Cat'
);

Distribution Plot displot()#

Used to plot the distribution of values. Extends the following axes plots:

  • histplot() (with kind=“hist”; the default)

  • kdeplot() (with kind=“kde”)

  • ecdfplot() (with kind=“ecdf”; univariate-only)

# change style 
sns.set_style('ticks')

# distribution plot 1. 1) kind='kde' then remove  kind='kde' 2) y='Defense' 3) rug= True 4) despine
sns.displot(data= pokemon, x='Attack', y='Defense', rug= True)
sns.despine(left= True, bottom= True);

# distribution plot 2.  1) hue='Stage' 2) kind='kde' , add multiple='stack' 3) multiple='dodge' 4) col='Speed Cat' 5) row='Stage' 
sns.displot(data= pokemon, x='Attack', hue='Stage', multiple='stack', col='Speed Cat', row='Stage');

Category plot catplot()#

Used to generate plots and charts with catgorical information. Extends the following categorical axes plots:

  • Categorical scatterplots:

    • stripplot() (with kind=“strip”; the default)

    • swarmplot() (with kind=“swarm”)

  • Categorical distribution plots:

    • boxplot() (with kind=“box”)

    • violinplot() (with kind=“violin”)

    • boxenplot() (with kind=“boxen”)

  • Categorical estimate plots:

    • pointplot() (with kind=“point”)

    • barplot() (with kind=“bar”)

    • countplot() (with kind=“count”)

keep_canvas = ['id', 'property_type', 'property_use', 'canvas_location', 'canvas_nature', 'viewing_potential']
keep_graffiti = ['canvas_id', 'type', 'message', 'width', 'height']

tbl =(
 canvas.loc[:,keep_canvas]
      .merge(graffiti[keep_graffiti], left_on='id', right_on='canvas_id', how='inner')
      .fillna({'message': 'none'})
      .drop(columns= ['id', 'canvas_id'])
      .assign(area= graffiti.width * graffiti.height)
)
